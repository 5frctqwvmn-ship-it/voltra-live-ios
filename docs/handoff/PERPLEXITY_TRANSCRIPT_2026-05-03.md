### 2. Chain-centric routing fix

User identified two bugs in the chain (chains overlay) routing:

**Bug A — Combined-mode chains overlay not split:**
- In `.combined` workoutMode, base weight was being split 50/50
  across both Voltras, but `upcomingChainsLb` was being sent to
  only one side.
- Fix: split chains evenly like base weight, rounded to nearest 2 lb.

**Bug B — Inverse chains payload incorrect:**
- Inverse chains (assist/accommodate resistance inversion) was using
  the same positive-value payload as normal chains.
- Firmware expects a negative value to signal assist direction.
- Fix: add `isInverseChains: Bool` flag to device state, send
  negative chains payload when true.
